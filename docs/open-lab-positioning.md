# Open Lab Positioning

`source-card-schema` supplies the source, claim, evidence, limitation, rights-note, and review-state primitive for the Hypernovelty Open Lab public footprint.

It supports:

- `agent-run-receipts` when an agent-assisted run needs compact source or evidence references.
- `ai-workflow-safety-screen` when a workflow decision depends on claims or source review.
- `hypernovelty-verification-literacy-kit` when learners need to separate claims from evidence.
- `ai-school-readiness-kit` when school AI use needs public-safe references.
- `hypernovelty-survival-kit` as the umbrella map for the public proof set.

## Public-Safe Boundary

A source card keeps provenance structured, but it does not prove a claim, clear rights, fetch sources, or decide publication safety. Public cards should be synthetic or reviewed for release and should not include private source relationships, local paths, credentials, account data, client material, or unpublished work.
